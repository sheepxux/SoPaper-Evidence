# Sopaper Evidence v1.0.0

## Summary

First full product release of Sopaper Evidence as an evidence-first research workflow skill.

## Highlights

- topic-first source discovery, fetch, conservative verification, and evidence-pack generation
- reviewed primary-source summaries beyond page-level metadata
- direct `.csv`, `.tsv`, and `.json` result-artifact ingestion
- multi-artifact fusion into aggregate `project_evidence`
- metric normalization for cleaner result evidence
- dedicated fairness review for comparative claims, baseline breadth, metric grounding, and scope alignment
- synchronized English and Chinese repository documentation
- synchronized GitHub and ClawHub release packaging

## Why this release matters

Sopaper Evidence now works as a real research evidence system rather than a prompt bundle. A user can start from a topic, bring in external sources and local experiment outputs, build a traceable evidence pack, judge comparative fairness, and keep downstream writing grounded in auditable support.

## Notes

- the workflow remains conservative by design
- fairness review is still a draft aid, not a substitute for expert judgment
- local `output/` test artifacts were not included in the release
