# Sopaper Evidence v0.6.0

## Summary

Topic-driven evidence workflow release with conservative source verification and structured result-artifact ingestion.

## Highlights

- added topic-first workflow coverage from search plan to source list, fetched source notes, ledger, claim map, and gap report
- improved external-source handling with conservative fetch and verification stages before evidence promotion
- strengthened benchmark, evaluation, baseline, and metric extraction for OpenClaw-style and retrieval-style topics
- added structured result-artifact ingestion so direct internal evidence can influence comparative claim support
- aligned README, skill metadata, and marketplace listing with the current evidence-discovery product direction

## Why this release matters

This release moves Sopaper Evidence beyond a drafting scaffold and into a usable evidence workflow. A user can now start from a research topic, discover candidate sources, turn them into reviewable evidence notes, combine them with internal result artifacts, and get a conservative evidence pack that clearly separates supported claims from remaining gaps.

## Notes

- external-source verification remains intentionally conservative and does not replace manual reading of primary sources
- comparative claims still depend on direct result evidence, baseline context, and scope review
- pipeline outputs remain drafts and should be reviewed before downstream writing
